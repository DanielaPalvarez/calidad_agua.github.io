Sistema de Alerta Temprana para la Calidad del Agua en Sonora

Detección Automatizada de Anomalías

Este proyecto tuvo como objetivo desarrollar un sistema automatizado de alerta temprana para detectar eventos atípicos en la calidad del agua en el estado de Sonora. Se buscó identificar posibles contaminaciones mediante técnicas de análisis de datos, detección de outliers y modelos de referencia, con el fin de apoyar a las autoridades y proteger a las comunidades potencialmente afectadas.

Comenzamos el proyecto con la recopilación y fusión de dos bases de datos fundamentales: los resultados de monitoreo de calidad del agua por sitio y la información geográfica de los sitios de muestreo. Estas bases fueron integradas utilizando la columna clave "CLAVE SITIO", permitiendo relacionar los resultados de laboratorio con su contexto geográfico (coordenadas, municipio, tipo de cuerpo de agua, etc.).

Posteriormente, se procedió a una fase de limpieza y depuración de datos. Dado que muchas variables no estaban presentes en todos los sitios debido a diferencias en los elementos analizados, se optó por no eliminar columnas con valores faltantes, ya que su ausencia era lógica según el sitio y el tipo de análisis realizado. Se evaluaron patrones de datos nulos por columna para respaldar esta decisión.

Para organizar el análisis, se diseñó una estrategia de clasificación de parámetros agrupándolos por categoría temática, como alcalinidad, bacteriológico, oxígeno, orgánico, nitrógeno y toxicidad. Esta clasificación se realizó cruzando las columnas del dataset con una tabla de simbología que describe cada parámetro, y posteriormente se aplicó un modelo de clustering KMeans para reforzar la agrupación por similitud de variables. Esto permitió la creación de subconjuntos temáticos para análisis independientes y detallados por tipo de contaminante.

Una vez segmentados los datos, se implementó una técnica robusta de detección de valores atípicos utilizando el rango intercuartílico (IQR). Para cada parámetro dentro de su respectivo subconjunto, se identificaron aquellas mediciones que caían fuera del rango aceptado, lo que fue interpretado como una posible alerta. Cada evento atípico fue clasificado y almacenado con su respectivo sitio, coordenadas, parámetro implicado, valor detectado, año y categoría.

Con los eventos atípicos identificados, se generó un sistema de alertas que consolidó todas las observaciones inusuales en un solo repositorio. Este sistema permite saber qué municipios, años y sitios han registrado alertas, así como el tipo de contaminante implicado. Esto sirvió como base para desarrollar visualizaciones interactivas y mapas que permiten monitorear la distribución espacial de las alertas y detectar patrones recurrentes o críticos.
